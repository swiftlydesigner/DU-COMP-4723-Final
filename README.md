# DU COMP 4723 Final Project
## Ghidra Reverse Engineering C Program

We are provided with a compiled binary containing intentional vulnerabilities and hidden features.

## Purpose of this repository
- Record notes
- Record screenshots and relevant data/snippets


## Objectives
Use Ghidra to perform static analysis of the binary. 


- Control flow analysis
- Data structure identification
- Function purpose reconstruction
- Memory corruption vulnerabilities
- Hidden functionality discovery

- Vulnerability Discovery (Minimum 5 required):
  - Identify and document at least five distinct vulnerabilities including:
  - Memory corruption issues (buffer overflows, format strings, etc.)
  - Privilege escalation paths
  - Information disclosure vulnerabilities
  - Cryptographic weaknesses
  - Time-based triggers
  - Environment manipulation vectors

- For each vulnerability, provide:
  - Location in the binary
  - Technical description
  - Potential impact
  - Exploitation methodology
  - Proof-of-concept (if applicable)

- Hidden Easter Egg Analysis:
   - Discover and document all hidden features and Easter eggs
   - Analyze the conditions required to trigger each feature
   - Explain the purpose and potential security implications of each hidden feature
 
## Deliverables
Through a video:
- Executive summary of findings
- Detailed technical analysis of each vulnerability
- Step-by-step discovery process
- Screenshots from Ghidra analysis
- Exploitation demonstrations (where applicable)
- Recommendations for mitigation
